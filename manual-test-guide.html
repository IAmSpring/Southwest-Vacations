Southwest Vacations - Date Selection Test Guide

This guide will help you test the date selection functionality on the trip details page and verify that selected dates flow correctly to the booking form.

Note: You need to have the application running locally for this test to work.

Manual Test Steps

Step 1: Navigate to a Trip Detail Page

Open the following URL in your browser:

http://localhost:5173/trip/trip1
Expected result: The Orlando, Florida trip details page should load with available dates displayed.

Step 2: Verify Default Date Selection

Check the list of available dates.

Expected result: The first date in the list should be selected by default (highlighted in blue).

Step 3: Select a Different Date

Click on a different date option (preferably the second date in the list).

Expected result:
  • The clicked date should become highlighted
  • The previously selected date should no longer be highlighted
  • The selected date should appear in the booking summary section

Step 4: Check the Book Now Button

Inspect the Book Now button URL (you can hover over it and check the status bar, or right-click and copy the link).

Expected result: The URL should include both the trip ID and the selected date as parameters, e.g., /book?trip=trip1&date=2023-06-22

Step 5: Navigate to the Booking Page

Click the Book Now button.

Expected result: You should be taken to the booking form page.

Step 6: Verify Date in Booking Form

Check that the selected date appears in the booking form.

Expected result:
  • The selected date should be displayed at the top of the form under "Selected Date:"
  • The departure date input field should be pre-filled with the selected date
  • The date input field should be highlighted with a blue border and light blue background

Step 7: Complete a Test Booking

Fill out the booking form with test data and submit.

Expected result: The form should submit successfully, and you should be redirected to a confirmation page.
